Pregunta 2 (10 puntos) -----------------------------------------------------------------------------
Dado un protocolo RDT que funciona sobre un canal sin pérdidas ni errores, mencione y justifique qué
mecanismos se deben agregar para soportar:
a) Un canal con errores de bit.
b) Un canal con errores de bit y pérdidas. Dibuje la máquina de estados del emisor para este último
caso.

a) Se debe agregar un mensaje de confirmacion/negacion ACK/NACK que el receptor envia al emisor 
avisando que el paquete que recibio de forma correcta. En si no se recibe de forma correcta se 
debe retrasmitir. Para manejar los paquetes duplicados se puede implementar un numero de secuencia
para identificar el paquete que se espera enviar/recibir.
b) Lo que falta agregar es un tiempo de espera maximo luego de enviar un paquete, si no se recibio 
una confirmacion del paquete se vuelve a enviar. Notar que si se pierde el paquete ACK igual se vuelve
a enviar nuevamente el paquete y el receptor debe volver a enviar el ACK al recibir el paquete 
duplicado.

Pregunta 3 (10 puntos) -----------------------------------------------------------------------------
a) Explique las características del modelo de capas que favorecen el desarrollo y actualización de las
redes de datos.
b) ¿Cómo se relacionan dos capas contiguas del modelo?
c) ¿Qué capas se presentan en el modelo Internet presentado en el curso y qué función cumple cada
una de ellas?

a) El modelo de capas favorecen el desarollo y actualizacion de las redes de datos porque abstraen 
la implementacion de la capa a una intefaz. Es decir la implementacion solo se debe de preocupar de 
satisfacer la intefaz (generalmente especificada en un RFC) y utilizar la interfaz de la capa inferior
es decir si se quiere implementar de una forma distinta, mientras que respete la intefaz 
automaticamente puede funcionar con el resto del ecosistema de las redes de datos.
b) Las capas se relacionan mediante una interfaz por lo general definida en un RFC.
c) El modelo Internet presentado en el curso presenta 5 capas:
Aplicacion: Utiliza la capa de transporte para implementar cualquier aplicacion de requiera de 
una comunicacion entre dos terminales en la red, estas aplicaciones son las que generalmente utiliza
el usuario comun.
Transporte: Se encarga de transportar mensajes entre procesos especificos entre terminales.
Red: Se encarga de dirigir los paquetes dentro de la red, crear la ruta que deben seguir.
Enlace: Se encarga de que el paquete llegue de el enlace actual al siguiente.
Fisica: Es el medio por donde se transmite el paquete de un enlace a otro.
