¿qué protocolos puede usar? Realice una breve explicación de cada uno y mencione qué 
diferencias hay entre ellos.

a) Se debe realizar una request DNS para el registro MX del dominio cables.uy, luego que se recibe

Pregunta 2 (8 puntos) ----------------------------------------------------------------------------
a) Suponga que un navegador web obtiene un objeto mediante el protocolo HTTP. ¿Cómo 
sabe el tipo de este objeto para representarlo correctamente en pantalla? Por ejemplo, 
cómo sabe si es una imagen jpeg, gif, o un texto?
b) Explique el mecanismo GET condicional comentando qué información se incluye para ello 
en la cabecera de HTTP, cómo es utilizada y cuál es el beneficio que se busca con su uso.

a) Se fija en la cabecera Content Type, que por ejemplo si el documento es de tipo HTML la cabecera
seria Content Type: text/html\r\n (notar que se incluye el retorno de carro y salto de linea para 
indicar el fin de la cabecera).
b) El GET condicional se utiliza para posiblilitar una implementacion de cache ya sea en un servidor 
proxy o mismo dentro del navegador, la idea es realizar un GET de un recurso y solo retornar el
recurso si fue modificado, en caso de no retornar el recurso se utiliza el guardado en cache, esto 
se utiliza para mejorar los tiempos de respuesta y reducir el uso de la red.

Pregunta 3 (10 puntos) ----------------------------------------------------------------------------
a) Enumere y explique dos ventajas y dos desventajas que percibe una aplicación al usar 
UDP como protocolo de transporte, en comparación con utilizar TCP.
b) Mencione una aplicación que utilice UDP como capa de transporte y a partir de su 
respuesta de la parte a), fundamente porque se podría haber tomado esa decisión de 
diseño.

a) Como ventajas estan: que no requiere de una conexion esto permite que el envio del mensaje sea 
mas veloz porque no requiere de los pasos previos a la conexion (1 RTT), tambien que la no tener 
control de congestion se tiene el control completo de en que momento se envia el mensaje.
Como desventajas estan el hecho de que no asegura que lleguen en orden ni que lleguen todos los 
paquetes, ademas el hecho de que no tenga control de congestion deja a manos del programador de 
la aplicacion controlar que no se genere la congestion.
b) Por ejemplo DNS utiliza UDP dado que prioriza la velocidad para enviar las consultas para resolver
los dominios, si usase TCP demoraria practicamente 3 veces mas porque ademas de la carga util de la 
consulta DNS se deberian de enviar los mensajes TCP de conexion SYN y SYNACK.
